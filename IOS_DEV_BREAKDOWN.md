# iOS 开发拆解文档（v0.1）

项目：Bazi Fortune（八字命理）
目标：将现有 Web MVP 打包为 iOS App，并逐步对齐 Figma（Polaris）视觉稿。

## 1) 产品流程（MVP）

1. 首页输入信息
   - 出生日期（必填）
   - 出生时间（必填）
   - 性别（可选）
   - 出生城市（可选）
2. 点击“开始分析”
3. 加载中状态
4. 结果页（结构化展示）
   - 性格特质
   - 运势（事业/学业）
   - 婚恋关系
   - 财富与消费
   - 规避事项
   - 行动建议
5. 错误页/失败提示

## 2) iOS 页面拆解（首批）

### P0（必须）
- `HomeInputView`：输入表单页
- `LoadingView`：分析中
- `ResultView`：结果展示
- `ErrorToast / ErrorView`：失败提示

### P1（增强）
- `HistoryView`：历史记录
- `SettingsView`：模型切换、隐私说明
- `ShareCardView`：结果卡片导出

## 3) 技术路线

### 路线A（最快上线）
Capacitor 包装 Web：
- 复用现有前端与后端 API
- 快速出 iOS 测试包（TestFlight）
- 后续逐屏替换为原生 UI

### 路线B（原生重构）
SwiftUI 全原生：
- 体验最好
- 开发周期更长

当前建议：先走 A，再逐步演进。

## 4) API 契约（当前）

- `POST /api/analyze`
- Request:
```json
{
  "birthDate": "1995-05-20",
  "birthTime": "13:30",
  "gender": "女",
  "city": "上海"
}
```
- Response:
```json
{
  "result": "...大模型返回文本..."
}
```

## 6) 开发任务拆分（可执行）

### 里程碑 M1（1-2 天）
- Capacitor 初始化 iOS 工程
- Web App 在 iPhone 模拟器可运行
- 首屏输入+结果完整跑通

### 里程碑 M2（2-4 天）
- 对齐 Figma 样式（P0 页面）
- 错误态、空态、加载态统一
- 隐私与免责声明完善

### 里程碑 M3（后续）
- 分享卡片
- 历史记录
- 模型切换与提示词模板

## 7) 风险与注意事项

- API Key 不可入前端包（仅服务端保存）
- 命理内容必须标注“仅供参考”
- 避免医疗/法律/投资确定性建议
- iOS 上架可能涉及内容审核，需准备合规说明
